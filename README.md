# TOTP 2FA Authentication Demo

## Overview

This project demonstrates the implementation of a Time-Based One-Time Password (TOTP) authentication mechanism using Python and the `pyotp` library.

The system generates a shared secret, creates a provisioning URI compatible with Google Authenticator, generates a QR code, and validates user-submitted TOTP codes.

This lab follows the TOTP standard defined in RFC 6238.

---

## Authentication Flow

Registration:

1. Generate a Base32 secret.
2. Create a TOTP object.
3. Generate an `otpauth://` provisioning URI.
4. Convert URI into a QR code.
5. User scans QR using Google Authenticator.

Authentication:

1. User enters 6-digit code.
2. Server verifies TOTP using shared secret.
3. Access decision is made.

---

## Secret Generation

```python
import pyotp

secret = pyotp.random_base32()
totp = pyotp.TOTP(secret)
```

The secret is shared between the server and the authenticator application.

---

## Provisioning URI and QR Code

Provisioning URI generation:

```python
uri = totp.provisioning_uri(name="user@example.com", issuer_name="SecureApp")
```

QR generation:

```python
import qrcode

qr = qrcode.make(uri)
qr.save("qrcode.png")
```

The QR code is scanned by Google Authenticator to register the account.

## Token Verification

The user submits a 6-digit code generated by the authenticator app.

```python
user_code = input("Enter OTP: ")

if totp.verify(user_code):
    print("Access granted")
else:
    print("Invalid token")
```

The verification is time-based and updates every 30 seconds.

## Security Considerations

- The secret must be stored securely server-side.
- Clock synchronization is important for correct validation.
- Token lifetime is limited to 30 seconds.
- TOTP reduces the risk of credential reuse and password compromise.
